## 2️⃣ Mode2 분할 매도 로직 완성

### 요구사항 정리
- **모수 = 현재 bought_quantity** (최초 매수량 아님!)
- 1차/2차 저항/지지 비중(%)은 **전체 보유 수량 기준**
- MTS 수동 매도 후 동기화 → 갱신된 bought_quantity 기준으로 재계산
- 설정된 모든 매도 레벨 소진 시 자동 모니터링 종료

### 구현 내용

**1. price_monitor.py - 매도 수량 계산 로직**
```python
# 현재 보유 수량(모수) 기준 비중 계산
current_bought_qty = watcher.get('bought_quantity', 0)
sell_qty = int(current_bought_qty * resistance_1_pct / 100)
```

**예시:**
```
매수: 100주 (bought_quantity=100)
설정: 1차저항 50%, 2차저항 30%

1차 저항 도달 → 100 × 50% = 50주 매도 (잔여 50주)
MTS에서 수동 10주 매도 → 동기화 → bought_quantity=40주
2차 저항 도달 → 40 × 30% = 12주 매도 (잔여 28주)
```

**2. mode2_manager.py - 매도 이력 추적**
- `sold_history` 배열에 매도한 레벨 기록
- 같은 레벨 중복 매도 방지
- 설정된 모든 레벨 소진 시 `active=false` 자동 설정

**3. 모니터링 종료 조건**
1. `bought_quantity = 0` (전량 매도)
2. 설정된 모든 매도 레벨 소진 (홀딩 포지션 진입)

### 결과

### 주요 이슈 해결

#### **Issue 1: 토큰 인증 실패**
**문제:** `_ensure_valid_token()` 반환값이 토큰 문자열이 아님
**해결:** `_get_token()` 메서드 추가하여 명시적으로 토큰 반환
```python
def _get_token(self):
    self.base_client._ensure_valid_token()
    return self.base_client.token
```

#### **Issue 2: JSON 직렬화 실패**
**문제:** NumPy `bool_` 타입이 JSON으로 직렬화 불가
**해결:** `convert_to_native_types()` 함수로 Python 네이티브 타입 변환
```python
def convert_to_native_types(obj):
    if isinstance(obj, (np.bool_, np.bool8)):
        return bool(obj)
    elif isinstance(obj, (np.integer, ...)):
        return int(obj)
    ...
```

#### **Issue 3: 연속조회 헤더 대소문자**
**문제:** 헤더 키 대소문자 불일치로 연속조회 실패
