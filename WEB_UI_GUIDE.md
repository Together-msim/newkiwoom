## 📝 종목 등록

### 필수 입력 항목 (상단 폼)

1. **종목코드** (예: 003280)
   - 6자리 숫자 또는 알파벳 포함 코드

2. **매수 타점** (원)
   - 매수하고자 하는 목표 가격
   - 예: 13000원

3. **Budget** (원)
   - 투자할 총 금액
   - 예: 300000원

4. **매수 수량** (자동계산)
   - `Budget / 매수타점`으로 자동 계산
   - 읽기 전용 필드

### 저항/지지 레벨 설정

#### 📈 저항 레벨 (익절)
- **2차 저항 가격**: 최고 목표가 (예: 17000원)
- **2차 익절 %**: 목표 수익률 (예: 30.8%)
- **1차 저항 가격**: 중간 목표가 (예: 15000원)
- **1차 익절 %**: 부분 익절 수익률 (예: 15.4%)

#### 📉 지지 레벨 (손절)
- **1차 지지 가격**: 첫 손절선 (예: 12000원)
- **1차 손절 %**: 손실률 (예: -7.7%)
- **2차 지지 가격**: 최종 손절선 (예: 11000원)
- **2차 손절 %**: 최대 손실률 (예: -15.4%)

### 저장
- 모든 값 입력 후 **저장** 버튼 클릭
- 테이블에 새 row가 추가됨

## 📊 감시 리스트 관리

### 테이블 컬럼
| 컬럼 | 설명 |
|------|------|
| 종목코드 | 정규화된 종목코드 |
| 종목명 | 종목 이름 (API 조회) |
| Mode | mode2 고정 |
| 매수타점 | 매수 목표 가격 |
| Budget | 투자 금액 |
| 수량 | 자동계산된 매수 수량 |
| 1차저항 | 1차 익절 가격 및 % |
| 2차저항 | 2차 익절 가격 및 % |
| 1차지지 | 1차 손절 가격 및 % |
| 2차지지 | 2차 손절 가격 및 % |
| 상태 | 매수대기중, 매도대기중, 자동매도완료, 수동매도완료 |
| 액션 | Active/Inactive/Delete 버튼 |

### Row 선택 (편집 모드)
1. 테이블에서 원하는 row 클릭
2. 상단 폼에 해당 종목의 데이터가 자동으로 채워짐
3. 값 수정 후 **수정** 버튼 클릭
4. **취소** 버튼으로 편집 모드 종료

### 액션 버튼
- **Active/Inactive**: 감시 활성화/비활성화 토글
  - Inactive 상태에서는 가격 모니터링 안 함
  - 테이블에서 흐리게 표시됨
- **X**: 종목 삭제
  - 확인 팝업 후 삭제

### 필터링
- **Active만 보기** 체크박스
  - 체크 시: Active=True인 종목만 표시
  - 미체크 시: 모든 종목 표시

## 🤖 자동매매 동작

### 1단계: 매수 시그널
- 현재가가 **매수 타점**의 ±1% 범위 내 도달
- 자동으로 지정가 매수 주문 실행
- 주문 수량: 자동계산된 수량
- 상태: `매수대기중` → `매도대기중`
- 텔레그램 알림 전송

### 2단계: 익절/손절 시그널
매도대기중 상태에서 아래 조건 체크 (우선순위 순):

#### 익절 (이익 실현)
1. **2차 저항** 도달 → 전량 매도
2. **1차 저항** 도달 → 부분 매도 (설정에 따라)

#### 손절 (손실 제한)
1. **2차 지지** 하락 → 전량 손절
2. **1차 지지** 하락 → 부분 손절 (설정에 따라)

### 체결 후
- 자동으로 지정가 매도 주문 실행
- 상태: `매도대기중` → `자동매도완료`
- 텔레그램 알림 전송 (매수가, 매도가, 손익률 포함)

## 🐛 문제 해결

### 웹 페이지가 열리지 않음
1. 웹 서버가 실행 중인지 확인
   ```bash
   # 터미널에 "웹 서버 시작: http://..." 메시지 확인
   ```
2. 포트 충돌 확인
   ```bash
   lsof -i :5000  # 5000번 포트 사용 중인 프로세스 확인
   ```

### 종목 저장 실패
1. 필수 필드 확인 (종목코드, 매수타점, Budget)
2. 종목코드 중복 확인 (이미 등록된 종목은 수정으로 처리)

### 가격 모니터링이 작동하지 않음
1. 키움 API 연결 확인
   ```bash
   python test_kiwoom.py
   ```
2. `.env`에 키움 API 설정 확인
3. `TELEGRAM_CHAT_ID` 설정 확인

### 자동매매가 실행되지 않음
- **현재 주문 API는 시뮬레이션 모드**
- 실제 주문을 위해서는 `kiwoom_client.py`의 `place_buy_order()`, `place_sell_order()` 메서드에 실제 키움 API 호출 로직 추가 필요

## 💡 팁

### 효율적인 사용법
1. **테스트 단계**
   - 소액 Budget으로 먼저 테스트
   - 저항/지지 레벨을 보수적으로 설정

2. **모니터링 주기**
   - 단기 매매: MONITOR_INTERVAL=5 (5초)
   - 스윙 매매: MONITOR_INTERVAL=30 (30초)

3. **리스크 관리**
   - 1차 지지를 매수가 대비 -5% 내외로 설정
   - 2차 지지를 최대 손실 허용 범위로 설정

4. **수익 최적화**
   - 1차 저항: 부분 익절로 리스크 감소
   - 2차 저항: 목표 수익률 달성
